Comment Metadata:

Metadata about your comments is stored when you copy or save them.
This includes a hash of your IP address, comment length and type of pattern used. This does not include class names, student names or comment text.

IP Addresses:

Cleartext IP address are logged for security when comments are rated, tagged or edited, or when an incorrect password is entered.
Hashed IP addresses are logged for analytics when you copy or save your comments.

Your Comments:

I do not save any finished comments, student names, class names or patterns.
The save and load file is done 100% in the browser. The server never sees those files.

Local Storage:

By default Comment Breeze will cache your comments, in your browser, in case something happens. This can be turned off in settings.
To clear this storage after you are done use the "Reset App" button in the Site Menu.

Cookies:

Cookies are not required to access Comment Breeze and no functionality will change if you disable them.
Comment Breeze does not set or use cookies, but Google Analytics does.

Email addresses:

Email addresses or names submitted on the contact form will never be shared or sold.
